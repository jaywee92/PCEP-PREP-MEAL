📚 Theory: Programming Fundamentals

Compiler vs Interpreter

Compiler Translates the entire source at once into an executable binary. End users run the result without needing the compiler. Examples: C, C++.
Interpreter Executes source code line by line at runtime — no separate compilation step visible to the user. Examples: early BASIC, shell scripts.
Python — Hybrid CPython compiles source to bytecode (.pyc), then the Python VM (PVM) interprets that bytecode. Users don't need the compiler — just the runtime.

Language Definition: Lexis · Syntax · Semantics

Lexis The set of valid words (tokens) a language accepts — keywords, identifiers, literals, operators.
Syntax Rules for combining words into valid sentences (statements). Describes grammatical structure.
Semantics The meaning of syntactically correct statements — what the program actually does when executed.

Operator Precedence (highest → lowest)

() › ** › +x  −x › *  /  //  % › +  − › comparisons › not › and › or 
print(-2 ** 2)      # -4   (**  before unary −: -(2**2))
print(2 ** 3 ** 2)  # 512  (right-to-left:  2**(3**2) = 2**9)

Syntax Rules — Identifiers & Keywords

Indentation Mandatory for all code blocks — no curly braces. Inconsistent indent → IndentationError.
Colon Required Every block header (if, for, def, …) ends with :. Missing it → SyntaxError.
Case-Sensitive name ≠ Name ≠ NAME. Python treats them as completely different identifiers.
Identifier Rules Must start with a letter or _. May contain letters, digits, _. No hyphens, spaces, or starting digits.
Reserved Words for, and, True, None, … are illegal as identifiers — eliminate these first on exam.

Input / Output — Full Pattern

# input() ALWAYS returns str — always cast before arithmetic
name     = input("Name: ")            # str
age      = int(input("Age: "))        # int
velocity = float(input("Speed: "))    # float

# String comparison is exact — spaces and case matter
print(name == " Robert ")             # False if user typed: robert

# print() keyword arguments
print("A", "B", sep="-", end="!")    # A-B!
score = 93.456
print(f"Score: {score:.2f}")         # Score: 93.46

Exam Strategy — Key Patterns

Type Conversion int(3.9)→3 (truncates), bool("0")→True (non-empty), bool("")→False (empty).
Operator Traps Add mental parentheses before computing. -2**2 = -(2**2) = -4, not 4.
Identifier Check Eliminate reserved words first, then check naming rules on remaining options.
Input Workflow Label every input() result as str immediately — cast before use.
String Equality Whitespace and case are part of the value. " Robert " ≠ "robert".
CPython Fact Default reference implementation, written in C. Compiles to bytecode; end users need only the runtime.
Compiler Fact Compilation produces a target artifact before runtime — end users don't install the compiler.
Question Formats Single-choice, multi-select, drag-and-drop, gap-fill — all appear on PCEP.

⚠️ Common Mistakes in Module 1

Mistake 1: Python 2 Print Syntax

print "Hello"   # SyntaxError in Python 3!
print("Hello")  # Correct

Mistake 2: Division Returns Float

5 / 2   # 2.5  — NOT 2 in Python 3
5 // 2  # 2    — floor division

Python 2: 5/2 = 2. Python 3: 5/2 = 2.5.

Mistake 3: Invalid Variable Names

123var  = 5   # starts with digit → SyntaxError
var-name = 10  # hyphen → SyntaxError
var name = 15  # space → SyntaxError
var123 = 5       # OK
_var_name = 10   # OK
varName = 15     # OK

Mistake 4: Type Confusion with input()

age = input("Age: ")   # str!
print(age + 5)          # TypeError
age = int(input("Age: "))
print(age + 5)           # 30

Mistake 5: Python is Purely Interpreted

Python compiles to bytecode (.pyc), then the PVM interprets it — a hybrid approach. CPython is the reference implementation.

Mistake 6: is Instead of ==

x = 256; y = 256
print(x is y)    # identity — not reliable for values
print(x == y)    # True — always use == for value comparison
print(x is None) # OK — use is for None checks

Mistake 7: Assuming Standard Rounding

print(round(2.5))  # 2 — banker's rounding!
print(round(3.5))  # 4

Mistake 8: Keywords as Identifiers

True = 1   # SyntaxError — True is a literal
for  = 10  # SyntaxError — reserved word

TRUE = 1         # legal
true_value = 10  # legal

Mistake 9: String Input Auto-Formats

name = input()
print(name == " Robert ")   # False for input: robert
# case + spaces must match exactly
name = input().strip().lower()
print(name == "robert")     # safe comparison

🚨 Critical Exam Pitfalls

The most common traps on the PCEP exam. Master these to avoid losing easy points!

🚨Division Always Returns Float
❌ Wrong: 10/2 = 5
✓ Correct: 10/2 = 5.0
/ always returns float. Use // for integer result.
🚨Floor Division with Negatives
❌ Wrong: -7//2 = -3
✓ Correct: -7//2 = -4
Floor division rounds toward negative infinity, not toward zero.
🚨int() Truncates, Not Rounds
❌ Wrong: int(3.9) = 4
✓ Correct: int(3.9) = 3
int() cuts the decimal — use round() to round.
🚨Exponentiation Before Negation
❌ Wrong: -2**2 = 4
✓ Correct: -2**2 = -4
** binds tighter than unary minus. -2**2 = -(2**2).
🚨Right-to-Left Exponentiation
❌ Wrong: 2**3**2 = 64
✓ Correct: 2**3**2 = 512
2**(3**2) = 2**9 = 512 (right-associative).
🚨bool() Truthiness Rules
❌ Wrong: bool('0') = False, bool([0]) = False
✓ Correct: bool('0') = True, bool([0]) = True
Non-empty strings and non-empty lists are always True, regardless of content.
🚨Operator Precedence
❌ Wrong: 5 + 3 * 2 = 16
✓ Correct: 5 + 3 * 2 = 11
* before +: 3×2=6, then 5+6=11.
🚨input() Returns String
❌ Wrong: age = input(); if age >= 18:
✓ Correct: age = int(input()); if age >= 18:
input() always returns str — cast with int() or float().
🚨print() Returns None
❌ Wrong: result = print('hello') has a value
✓ Correct: print() returns None — side effect only
Use return to pass values from functions, not print().
🚨String + Number TypeError
❌ Wrong: x = '3' + 3
✓ Correct: x = int('3') + 3 OR str(3) + '3'
Cannot concatenate str and int — convert both to the same type first.
🚨Banker's Rounding
❌ Wrong: round(2.5) = 3
✓ Correct: round(2.5) = 2, round(3.5) = 4
Python rounds .5 to the nearest even number — a common exam trap.
🚨Float Precision Is Not Exact
❌ Wrong: 0.1 + 0.2 == 0.3 is True
✓ Correct: 0.1 + 0.2 == 0.3 # False
Binary approximation: floating-point equality checks can fail unexpectedly.
🚨Keyword vs Identifier
❌ Wrong: True, for, and are valid variable names
✓ Correct: TRUE, true_value are valid; True/for/and are illegal
Similar spelling ≠ same legality. Keywords and literals are off-limits as identifiers.
🚨String Equality Is Exact
❌ Wrong: " Robert " == "robert"
✓ Correct: Case and spaces must match exactly
Normalize with strip() and lower() before comparison when needed.
