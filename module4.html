Function Definitions & Default Arguments

def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

print(greet("Alice"))           # Hello, Alice!
print(greet("Bob", "Hi"))       # Hi, Bob!

# Calling pattern — all three are valid for def f(x=0):
f()       # uses default  → 0
f(7)      # positional    → 7
f(x=9)    # keyword       → 9

*args and **kwargs

def func(*args):          # tuple of positional args
    for arg in args:
        print(arg)

def func(**kwargs):       # dict of keyword args
    for key, val in kwargs.items():
        print(key, val)

func(1, 2, 3)             # *args → (1, 2, 3)
func(name="Alice", age=30) # **kwargs → {'name':'Alice','age':30}

Parameter order rule: positional → *args → keyword-only / default → **kwargs

Exception Handling — try / except / else / finally

try:
    num = int("abc")
except ValueError:          # specific first
    print("Invalid number!")
except Exception:           # general last
    print("Other error!")
else:
    print(f"Success: {num}")  # runs only if no exception
finally:
    print("Done!")            # ALWAYS runs

finally executes even when return or break appear inside try. A return inside finally overrides any earlier return.

Common Exceptions

ValueError
Right type, wrong value — e.g. int("abc")
TypeError
Wrong type for operation — e.g. None + 1, int_val[0]
KeyError
Dict key not found — d["missing"]
IndexError
Sequence index out of range — lst[99]
ZeroDivisionError
Division or modulo by zero — 5 / 0
AttributeError
Object has no such attribute — None.upper()

Recursion, Generators & Return-Type Safety

# Recursion — base case is required
def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n - 1)

# Generator — lazy value production via yield
def count_up_to(n):
    i = 1
    while i <= n:
        yield i
        i += 1

print(list(count_up_to(4)))   # [1, 2, 3, 4]

# Return-type safety — always verify what a function returns
# before indexing or arithmetic operations
def process(x):
    return x + 1        # returns int

value = process(3)      # 4  (int, NOT subscriptable)
# value[0]             # TypeError: 'int' object is not subscriptable

def combine(a, b):
    print(a + b)        # no explicit return → None

result = combine(2, 3)
# result[0]            # TypeError: 'NoneType' object is not subscriptable

Exam Strategy — Key Patterns

Call-Validation Rule Check required params first; positional args must come before keyword args. Count required params vs supplied args before evaluating logic.
Return Rule Once return executes, the function exits immediately — code after it is unreachable.
None Implicit Return Functions without explicit return always return None. None + 1 → TypeError.
Return-Type Rule Verify function return type before subscripting or arithmetic. int/None are not subscriptable.
Exception Rule Order handlers from specific → general; finally always runs regardless of exceptions or returns.
Arity Rule Match argument count exactly with required parameters before evaluating expression logic.
Generator Rule list(generator_obj) materializes lazy values. Generators cannot be rewound automatically.
Scope Rule Assignment inside a function creates a local — use global only when assigning to module-level names.
Multi-Select Validity Exam often asks which calls are valid for signatures with required + default params. Test each option independently.
Immutable Rebind vs Mutable Mutation Passing int/str: rebinding inside function does NOT affect caller. Passing list: in-place mutation DOES affect caller.

⚠️ Common Mistakes with Functions & Exceptions

Mistake 1: Mutable Default Arguments

Wrong:

def add_item(item, items=[]):
    items.append(item)
    return items

print(add_item(1))  # [1]
print(add_item(2))  # [1, 2] - WRONG!

Correct:

def add_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items

Mistake 2: Forgetting global Keyword

Wrong:

x = 10
def func():
    x = 20  # Creates local x
func()
print(x)  # Still 10, not 20!

Correct:

x = 10
def func():
    global x
    x = 20
func()
print(x)  # 20

Mistake 3: finally with return

Problem: finally return overwrites try return!

def func():
    try:
        return "try"
    finally:
        return "finally"  # This overwrites!

print(func())  # "finally" - WRONG!

Avoid return in finally block!

Mistake 4: Wrong Exception Order

Wrong:

try:
    code
except Exception:          # Catches all
    pass
except ValueError:        # Never reached!
    pass

Correct:

try:
    code
except ValueError:        # Specific first
    pass
except Exception:         # General last
    pass

Mistake 5: Not Catching Specific Exceptions

Wrong:

try:
    x = int(input())
except:  # Bare except catches everything!
    print("Error")

Correct:

try:
    x = int(input())
except ValueError:
    print("Invalid number")

Mistake 6: Assuming Functions Return Last Expression

def f():
    x = 5

print(f())   # None, not 5

Mistake 7: No Base Case in Recursion

def bad(n):
    return bad(n - 1)   # RecursionError eventually

Mistake 8: Treating None as Numeric Value

x = None
# print(x + 1)   # TypeError
print(x is None) # correct check

Mistake 9: Indexing Function Results Without Type Check

def process(x):
    return x + 1

val = process(3)      # int
# print(val[0])       # TypeError

🚨 Critical Exam Pitfalls

These are the most common traps that catch even experienced programmers in the PCEP exam. Master these to avoid losing easy points!

🚨 Mutable Default Arguments
❌ Wrong: def f(x=[]): x.append(1); return x # Works correctly
✓ Correct: def f(x=None): if x is None: x = []; x.append(1); return x # Correct pattern
Mutable defaults (like lists) are created ONCE at function definition. All calls share the same object. This is a classic Python gotcha. Use None as default and initialize inside.
🚨 print() vs return
❌ Wrong: result = print('answer: 42')
✓ Correct: print('answer: 42'); return 42
print() displays output but returns None. To return a value from a function, use return.
🚨 Forgetting Parentheses
❌ Wrong: func instead of func()
✓ Correct: func() calls the function
Without parentheses, func is a reference to the function object itself. You must use func() to actually call/execute it.
🚨 Global Keyword for Modification
❌ Wrong: x = 10; def f(): x = 20 # Changes local x
✓ Correct: x = 10; def f(): global x; x = 20 # Changes global x
Assignment creates a local variable unless you use the global keyword. You only need it if you're ASSIGNING; reading works fine without it.
🚨 Exception Order Matters
❌ Wrong: except Exception: ... except ValueError: ...
✓ Correct: except ValueError: ... except Exception: ...
Place specific exceptions BEFORE general ones. General exceptions catch everything that follows, making later except blocks unreachable.
🚨 finally Overrides return
❌ Wrong: def f(): try: return 1; finally: return 2 # Returns 1
✓ Correct: def f(): try: return 1; finally: return 2 # Returns 2
A return statement in finally overrides any return from try or except blocks. Use finally for cleanup, not returns.
🚨 finally Always Executes
❌ Wrong: finally doesn't run if there's a return
✓ Correct: finally ALWAYS runs, even with return, break, or exception
The finally block is guaranteed to execute before the function exits or the loop breaks, making it perfect for cleanup code.
🚨 Lambda Limitations
❌ Wrong: lambda x: x += 1
✓ Correct: lambda x: x + 1 # Only expressions allowed
Lambda functions can only contain a single expression, not statements. No assignments, loops, or multi-line code. For complex logic, use def.
🚨 *args is Tuple, **kwargs is Dict
❌ Wrong: *args is a list of positional arguments
✓ Correct: *args is a TUPLE of positional arguments; **kwargs is a dict
*args packs positional arguments into a tuple. **kwargs packs keyword arguments into a dictionary. Tuples are immutable.
🚨 Function Without return
❌ Wrong: def f(): x = 5 # Returns 5
✓ Correct: def f(): x = 5 # Returns None
If a function doesn't have an explicit return statement, it returns None. This is often the source of subtle bugs.
🚨 Bare except Hides Useful Errors
❌ Wrong: except: print("Error")
✓ Correct: except ValueError as e: ...
Catching everything masks root causes and can hide bugs. Prefer specific exceptions first.
🚨 None in Arithmetic
❌ Wrong: None + 1 is valid because None means 0
✓ Correct: None + 1 raises TypeError
Raw exam statements often mix true/false claims about None. None is assignable and comparable, but not valid with arithmetic operators.
🚨 Subscript on Scalar Return Values
❌ Wrong: process(3)[0]
✓ Correct: Only subscript sequences; int/None returns are not indexable
New appendix-style items use this frequently. Always determine function return type before calling index/slice operations.
