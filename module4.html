*args and **kwargs

def func(*args):
    for arg in args:
        print(arg)

def func(**kwargs):
    for key, val in kwargs.items():
        print(key, val)

func(1, 2, 3)                  # *args as tuple
func(name="Alice", age=30)     # **kwargs as dict

Variable Scope

x = 10  # Global

def func():
    y = 20  # Local
    global x
    x = 15
    return y

func()
print(x)  # 15

Exception Handling

try:
    num = int("abc")
except ValueError:
    print("Invalid number!")
except Exception:
    print("Other error!")
else:
    print(f"Success: {num}")
finally:
    print("Done!")

‚ö†Ô∏è Common Mistakes with Functions & Exceptions

Mistake 1: Mutable Default Arguments

Wrong:

def add_item(item, items=[]):
    items.append(item)
    return items

print(add_item(1))  # [1]
print(add_item(2))  # [1, 2] - WRONG!

Correct:

def add_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items

Mistake 2: Forgetting global Keyword

Wrong:

x = 10
def func():
    x = 20  # Creates local x
func()
print(x)  # Still 10, not 20!

Correct:

x = 10
def func():
    global x
    x = 20
func()
print(x)  # 20

Mistake 3: finally with return

Problem: finally return overwrites try return!

def func():
    try:
        return "try"
    finally:
        return "finally"  # This overwrites!

print(func())  # "finally" - WRONG!

Avoid return in finally block!

Mistake 4: Wrong Exception Order

Wrong:

try:
    code
except Exception:          # Catches all
    pass
except ValueError:        # Never reached!
    pass

Correct:

try:
    code
except ValueError:        # Specific first
    pass
except Exception:         # General last
    pass

Mistake 5: Not Catching Specific Exceptions

Wrong:

try:
    x = int(input())
except:  # Bare except catches everything!
    print("Error")

Correct:

try:
    x = int(input())
except ValueError:
    print("Invalid number")

Mistake 6: Assuming Functions Return Last Expression

def f():
    x = 5

print(f())   # None, not 5

Mistake 7: No Base Case in Recursion

def bad(n):
    return bad(n - 1)   # RecursionError eventually

üö® Critical Exam Pitfalls

These are the most common traps that catch even experienced programmers in the PCEP exam. Master these to avoid losing easy points!

üö® Mutable Default Arguments
‚ùå Wrong: def f(x=[]): x.append(1); return x # Works correctly
‚úì Correct: def f(x=None): if x is None: x = []; x.append(1); return x # Correct pattern
Mutable defaults (like lists) are created ONCE at function definition. All calls share the same object. This is a classic Python gotcha. Use None as default and initialize inside.
üö® print() vs return
‚ùå Wrong: result = print('answer: 42')
‚úì Correct: print('answer: 42'); return 42
print() displays output but returns None. To return a value from a function, use return.
üö® Forgetting Parentheses
‚ùå Wrong: func instead of func()
‚úì Correct: func() calls the function
Without parentheses, func is a reference to the function object itself. You must use func() to actually call/execute it.
üö® Global Keyword for Modification
‚ùå Wrong: x = 10; def f(): x = 20 # Changes local x
‚úì Correct: x = 10; def f(): global x; x = 20 # Changes global x
Assignment creates a local variable unless you use the global keyword. You only need it if you're ASSIGNING; reading works fine without it.
üö® Exception Order Matters
‚ùå Wrong: except Exception: ... except ValueError: ...
‚úì Correct: except ValueError: ... except Exception: ...
Place specific exceptions BEFORE general ones. General exceptions catch everything that follows, making later except blocks unreachable.
üö® finally Overrides return
‚ùå Wrong: def f(): try: return 1; finally: return 2 # Returns 1
‚úì Correct: def f(): try: return 1; finally: return 2 # Returns 2
A return statement in finally overrides any return from try or except blocks. Use finally for cleanup, not returns.
üö® finally Always Executes
‚ùå Wrong: finally doesn't run if there's a return
‚úì Correct: finally ALWAYS runs, even with return, break, or exception
The finally block is guaranteed to execute before the function exits or the loop breaks, making it perfect for cleanup code.
üö® Lambda Limitations
‚ùå Wrong: lambda x: x += 1
‚úì Correct: lambda x: x + 1 # Only expressions allowed
Lambda functions can only contain a single expression, not statements. No assignments, loops, or multi-line code. For complex logic, use def.
üö® *args is Tuple, **kwargs is Dict
‚ùå Wrong: *args is a list of positional arguments
‚úì Correct: *args is a TUPLE of positional arguments; **kwargs is a dict
*args packs positional arguments into a tuple. **kwargs packs keyword arguments into a dictionary. Tuples are immutable.
üö® Function Without return
‚ùå Wrong: def f(): x = 5 # Returns 5
‚úì Correct: def f(): x = 5 # Returns None
If a function doesn't have an explicit return statement, it returns None. This is often the source of subtle bugs.
üö® Bare except Hides Useful Errors
‚ùå Wrong: except: print("Error")
‚úì Correct: except ValueError as e: ...
Catching everything masks root causes and can hide bugs. Prefer specific exceptions first.
