📚 Theory: Data Collections

Strings

Strings are sequences of characters. They are immutable (cannot be changed).

text = "Hello, World!"
print(text[0])          # 'H'
print(text[-1])         # '!'
print(text[0:5])        # 'Hello'
print(text.upper())     # 'HELLO, WORLD!'
print(text.lower())     # 'hello, world!'
print(len(text))        # 13

Lists

Lists are ordered, mutable collections. They can contain any data type.

numbers = [1, 2, 3, 4, 5]
numbers.append(6)           # Add element
numbers.remove(3)           # Remove by value
numbers.pop()               # Remove last
numbers.extend([7, 8])      # Add multiple
print(numbers[0])           # 1
print(len(numbers))         # Length

Tuples

Tuples are ordered, immutable collections. Once created, they cannot be changed.

coords = (10, 20)
print(coords[0])            # 10
x, y = coords               # Unpacking
print(len(coords))          # 2

# Tuples are faster and safer than lists!

Dictionaries

Dictionaries are unordered, mutable collections with key-value pairs.

person = {"name": "Alice", "age": 30}
print(person["name"])       # Alice
person["age"] = 31          # Modify
person["city"] = "NYC"      # Add
del person["city"]          # Delete
print("name" in person)     # True
print(person.keys())        # Keys only

Common Methods

  • .append() - Add to list
  • .remove() - Remove by value
  • .pop() - Remove last or by index
  • .sort() - Sort list in-place
  • .reverse() - Reverse list
  • .count() - Count occurrences
  • .index() - Find index of value

Slicing Rules That Drive Many Exam Questions

s = "python"
print(s[1:4])   # yth (stop excluded)
print(s[::-1])  # nohtyp
print(s[100:200])  # "" (no IndexError for slicing)

Copy vs Reference Behavior

a = [1, 2, 3]
b = a           # same object
c = a[:]        # copy
b.append(4)
print(a)        # [1, 2, 3, 4]
print(c)        # [1, 2, 3]

Mutation Pattern Seen in New Appendix Questions

nums = [1, 2, 3]
vals = nums
del vals[1:2]
print(nums)   # [1, 3]

# deletion through alias mutates the same underlying list

Exam-Simulator Patterns for Module 3

  • String tasks: indexing/slicing, immutability, find() vs index().
  • List tasks: append() vs extend(), copy/reference, alias side effects, and sort() behavior.
  • Dict tasks: key checks with in, safe access with get().
  • Tuple tasks: single-item tuple syntax, unpacking count errors, and immutability (append()/pop() are unavailable).
  • Slicing targets: excluding trailing records with seq[:-5] is a common exam pattern.
  • Immutability traps: operations like del text[i] fail on strings with TypeError.
  • Alias check: when you see b = a, assume shared object unless explicit copy is shown.
  • Slice check: always verify stop index is excluded and negative indexes count from the end.
  • Dictionary check: remember key in d tests keys only, never values.
  • Key-type check: "2" and 2 are different dictionary keys.
  • Mutation check: evaluate in-place operations (del, append, pop) against aliasing first.

⚠️ Common Mistakes in Data Collections

Mistake 1: Forgetting Strings Are Immutable

Wrong:

text = "Hello"
text[0] = "J"  # TypeError!

Correct:

text = "Hello"
text = "J" + text[1:]  # Create new string

Mistake 2: Off-by-One Slicing

Problem: The upper bound is exclusive!

lst = [0, 1, 2, 3, 4]
lst[1:3]  # [1, 2], NOT [1, 2, 3]!

Remember: stop index is exclusive!

Mistake 3: Modifying List While Iterating

Wrong:

lst = [1, 2, 3, 4]
for x in lst:
    lst.remove(x)  # Unpredictable!

Correct:

lst = [1, 2, 3, 4]
for x in lst[:]:      # Iterate over copy
    lst.remove(x)

Mistake 4: List Copy Issues

Problem: = creates reference, not copy!

a = [1, 2, 3]
b = a
b.append(4)  # a = [1, 2, 3, 4] !

Solution:

a = [1, 2, 3]
b = a.copy()      # or list(a)
b.append(4)       # a unchanged

Mistake 5: Dictionary KeyError

Wrong:

d = {"a": 1}
print(d["b"])  # KeyError!

Correct:

d = {"a": 1}
print(d.get("b", "default"))  # "default"

Mistake 6: Confusing append() and extend()

Wrong expectation:

a = [1, 2]
a.append([3, 4])   # [1, 2, [3, 4]]

Correct alternative:

a = [1, 2]
a.extend([3, 4])   # [1, 2, 3, 4]

Mistake 7: Single-Element Tuple Without Comma

x = (1)     # int
y = (1,)    # tuple

Mistake 8: Assuming Alias Variables Are Copies

nums = [1, 2, 3]
vals = nums
vals.pop()
print(nums)   # [1, 2], not [1, 2, 3]

Mistake 9: Mixing Key Types in Dictionaries

Trap: numeric and string keys are distinct.

d = {2: "A"}
print("2" in d)  # False
print(d.get("2"))  # None

🚨 Critical Exam Pitfalls

These are the most common traps that catch even experienced programmers in the PCEP exam. Master these to avoid losing easy points!

🚨 Strings are Immutable
❌ Wrong: s = 'hello'; s[0] = 'H'
✓ Correct: s = 'hello'; s = 'H' + s[1:] OR s = s.replace('h', 'H')
Strings cannot be modified in place. You must create a new string. Lists ARE mutable and can be modified with index assignment.
🚨 List Assignment Creates Reference
❌ Wrong: b = a creates a new list
✓ Correct: b = a shares the same object
Assignment doesn't copy the list, it creates another reference to the same object. Modifying b also modifies a.
🚨 Copying Lists Requires Special Syntax
❌ Wrong: b = a copies the list
✓ Correct: b = a[:] OR b = a.copy() OR b = list(a)
To create an independent copy, use slicing a[:], .copy(), or the list() constructor.
🚨 sort() Returns None
❌ Wrong: b = a.sort()
✓ Correct: a.sort() OR b = sorted(a)
The .sort() method sorts in-place and returns None. Use sorted() if you need a new sorted list.
🚨 append() vs extend()
❌ Wrong: a.append([1,2]) adds items 1 and 2
✓ Correct: a.append([1,2]) adds the list [1,2] as single item; a.extend([1,2]) adds items
append() adds the argument as a single item. extend() adds each item from the iterable separately.
🚨 Single Element Tuple Syntax
❌ Wrong: t = (1) creates a tuple
✓ Correct: t = (1,) creates a tuple; (1) is just int
A single value in parentheses is just that value. You MUST use a trailing comma to create a single-element tuple: (1,)
🚨 Dictionary in Checks Keys
❌ Wrong: 'value' in d checks if 'value' is a value
✓ Correct: 'value' in d checks if 'value' is a key
The in operator for dictionaries checks keys only, not values. Use 'value' in d.values() to check values.
🚨 find() vs index()
❌ Wrong: s.index('x') returns -1 if not found
✓ Correct: s.find('x') returns -1; s.index('x') raises ValueError
find() returns -1 if substring not found. index() raises a ValueError. Choose based on your error handling needs.
🚨 String Slicing Never Errors
❌ Wrong: s[100:200] raises IndexError
✓ Correct: s[100:200] returns empty string if out of bounds
String slicing (and list slicing) is forgiving - out-of-bounds indices don't raise errors, they just return a shorter slice or empty result.
🚨 List Multiplication with Nested Objects
❌ Wrong: [[0]*3]*3 creates 3 independent rows
✓ Correct: [[0]*3]*3 shares the same inner list
Multiplying a list doesn't create independent copies. All rows reference the same inner list. Use a list comprehension instead: [[0]*3 for _ in range(3)]
🚨 Modifying Dict While Iterating
❌ Wrong: for k in d: d[k] = v works fine
✓ Correct: for k in d: d[k] = v raises RuntimeError
You cannot add or remove keys from a dictionary while iterating over it. Iterate over a copy: for k in list(d): or for k in d.keys():
🚨 Dict get() vs Bracket Notation
❌ Wrong: d['key'] returns None if missing
✓ Correct: d.get('key') returns None; d['key'] raises KeyError
dict.get(key) safely returns None for missing keys. Bracket notation d[key] raises KeyError. Use get() for safe access.
🚨 Tuple Can Contain Mutable Elements
❌ Wrong: Nothing inside a tuple can change
✓ Correct: (1, 2, [3, 4]) allows list mutation
Tuples are immutable containers, but mutable objects inside them can still be modified.
🚨 Aliasing After Assignment
❌ Wrong: vals = nums creates a separate list
✓ Correct: vals = nums creates a second reference to same list
Raw exam tasks often hide this in short snippets. Any in-place change through one name appears through the other name too.
🚨 Dictionary Key Type Mismatch
❌ Wrong: d[2] and d["2"] access the same key
✓ Correct: 2 and "2" are different keys and can map to different values
This appears in multiple imported questions. Check key type carefully before membership tests or lookups.
