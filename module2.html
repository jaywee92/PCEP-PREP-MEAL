Conditional Statements (if/elif/else)

Conditional statements allow code to be executed only under certain conditions. The basic syntax is:

if condition:
    # Code executes if condition is True
elif other_condition:
    # Code executes if other_condition is True
else:
    # Code executes if no condition is True

Comparison Operators

The most important comparison operators in Python:

  • == - Equality
  • != - Inequality
  • < - Less than
  • > - Greater than
  • <= - Less than or equal
  • >= - Greater than or equal

Boolean Logic

The operators and, or, and not are used to create complex conditions:

if age >= 18 and age < 65:
    print("You are working age")

if status == "student" or status == "teacher":
    print("You are in education")

if not admin:
    print("You are not an administrator")

for Loops

The for loop iterates over a sequence:

for i in range(5):
    print(i)  # Prints 0, 1, 2, 3, 4

for name in ["Alice", "Bob", "Charlie"]:
    print(f"Hello, {name}!")

while Loops

The while loop runs as long as a condition is met:

x = 0
while x < 5:
    print(x)
    x += 1

Loop Control: break and continue

break exits the loop immediately, continue skips to the next iteration:

for i in range(10):
    if i == 5:
        break  # Exits the loop
    if i % 2 == 0:
        continue  # Skips to next step
    print(i)

Truthiness and Boolean Evaluation

if []:
    print("won't run")
if "0":
    print("runs")         # non-empty string is truthy

print(5 and 3)            # 3
print(0 or 9)             # 9

range() Semantics (Python Docs Aligned)

range(5)          # 0..4
range(2, 7)       # 2..6
range(10, 0, -2)  # 10, 8, 6, 4, 2

# start + step*i must stay below stop (or above for negative step)

Loop else and Nested break

for i in range(3):
    if i == 1:
        break
else:
    print("no break")   # skipped because break happened

for i in range(2):
    for j in range(2):
        break           # only inner loop breaks

‚ö†Ô∏è Common Mistakes in Control Flow

Mistake 1: Assignment instead of Comparison

Wrong:

if x = 5:  # SyntaxError!
    print("Yes")

Correct:

if x == 5:  # Comparison
    print("Yes")

Mistake 2: Missing Colon

Wrong:

if x > 5
    print("Greater")  # IndentationError!

Correct:

if x > 5:
    print("Greater")

Mistake 3: Misunderstanding range()

Problem: range(5) goes from 0 to 4, NOT to 5!

for i in range(5):
    print(i)  # Prints 0, 1, 2, 3, 4 (not 5!)

The upper bound is EXCLUSIVE!

Mistake 4: else in Loops Misunderstood

Problem: The else clause is only executed if the loop ends WITHOUT break!

for i in range(5):
    if i == 3:
        break
else:
    print("Done")  # This will NOT print!

The else is NOT executed because of break!

Mistake 5: Infinite Loops

Problem: A while loop without an exit condition runs forever!

x = 0
while x < 10:
    print(x)
    # Mistake: x never increases!
    # Loop runs forever...

Don't forget to update the loop variable!

Mistake 6: Expecting break to Exit All Loops

Trap: break exits only the innermost loop.

for i in range(2):
    for j in range(2):
        break
    print(i)  # still runs for each outer iteration

Mistake 7: Over-Comparing to True/False

Less Pythonic:

if flag == True:
    ...
if len(items) != 0:
    ...

Preferred:

if flag:
    ...
if items:
    ...

Mistake 8: Assuming continue Cancels Loop else

for i in range(3):
    if i == 1:
        continue
else:
    print("runs")  # runs, because no break

üö® Critical Exam Pitfalls

These are the most common traps that catch even experienced programmers in the PCEP exam. Master these to avoid losing easy points!

üö® Loop else Clause Condition
‚ùå Wrong: else runs every time loop ends
‚úì Correct: else runs only if NO break occurred
The else block in a loop only executes if the loop completes normally. If break is executed, the else is skipped.
üö® range() Stop is Exclusive
‚ùå Wrong: range(5) = [0,1,2,3,4,5]
‚úì Correct: range(5) = [0,1,2,3,4]
The stop value in range() is exclusive - it's not included. range(5) goes from 0 to 4, not 5.
üö® Wrong Step Direction
‚ùå Wrong: range(5, 0, 1) returns [1,2,3,4]
‚úì Correct: range(5, 0, 1) returns []
With a positive step, you can't go from 5 down to 0. Use a negative step: range(5, 0, -1) for [5,4,3,2,1].
üö® break Only Exits Innermost Loop
‚ùå Wrong: break from nested loop exits all loops
‚úì Correct: break only exits the innermost loop
A break statement only terminates the loop it's in. For nested loops, you need multiple breaks or a flag variable.
üö® Chained Comparisons are Valid
‚ùå Wrong: 1 < x < 10 is not valid Python
‚úì Correct: 1 < x < 10 is valid and equivalent to (1 < x) and (x < 10)
Python supports chained comparisons, which are both valid and more efficient than writing separate conditions.
üö® Boolean Operators Return Values
‚ùå Wrong: 5 and 3 = True, 0 or 5 = True
‚úì Correct: 5 and 3 = 3, 0 or 5 = 5
and and or return actual values, not booleans. and returns the first falsy value or the last, or returns the first truthy value or the last.
üö® Assignment vs Comparison Operator
‚ùå Wrong: if x = 5: ...
‚úì Correct: if x == 5: ... OR x = 5; if x: ...
Use == for comparison and = for assignment. In conditions, = is a syntax error in Python.
üö® Loop Variable Persists After Loop
‚ùå Wrong: i is undefined after loop ends
‚úì Correct: i remains accessible after loop with its final value
Loop variables don't have block scope in Python. The variable i continues to exist and holds the last value it had.
üö® while True Without Break
‚ùå Wrong: while True continues until function ends
‚úì Correct: while True creates infinite loop - must have break
A while True loop will run forever unless a break statement is executed. Always ensure an exit condition.
üö® elif vs Multiple if Statements
‚ùå Wrong: All conditions are checked and all matching blocks run
‚úì Correct: Only the first true condition executes with elif
With elif, only the first matching block executes. With multiple if statements, all conditions are checked and all matching blocks run.
üö® all([]) and any([])
‚ùå Wrong: all([]) is False and any([]) is True
‚úì Correct: all([]) is True and any([]) is False
Empty iterables are a common logic trap. Learn these two special cases for exam questions.
üö® range() Is Not a List
‚ùå Wrong: type(range(5)) is list
‚úì Correct: type(range(5)) is range
range() returns a range object (lazy sequence), not a concrete list.
üö® Precedence in Mixed Boolean Conditions
‚ùå Wrong: and and or have equal precedence
‚úì Correct: and binds tighter than or
Raw exam items repeatedly test this rule. Without parentheses, Python evaluates and before or.
