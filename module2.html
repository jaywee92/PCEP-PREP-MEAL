📚 Theory: Control Flow

Conditional Statements (if/elif/else)

Conditional statements allow code to be executed only under certain conditions. The basic syntax is:

if condition:
    # Code executes if condition is True
elif other_condition:
    # Code executes if other_condition is True
else:
    # Code executes if no condition is True

Comparison Operators

==Equality — checks if two values are equal.
!=Inequality — checks if two values are not equal.
<Less than — left side is smaller.
>Greater than — left side is larger.
<=Less than or equal — left side is at most right side.
>=Greater than or equal — left side is at least right side.

Boolean Logic

The operators and, or, and not are used to create complex conditions:

if age >= 18 and age < 65:
    print("You are working age")

if status == "student" or status == "teacher":
    print("You are in education")

if not admin:
    print("You are not an administrator")

for Loops

The for loop iterates over a sequence:

for i in range(5):
    print(i)  # Prints 0, 1, 2, 3, 4

for name in ["Alice", "Bob", "Charlie"]:
    print(f"Hello, {name}!")

while Loops

The while loop runs as long as a condition is met:

x = 0
while x < 5:
    print(x)
    x += 1

Loop Control: break and continue

break exits the loop immediately, continue skips to the next iteration:

for i in range(10):
    if i == 5:
        break  # Exits the loop
    if i % 2 == 0:
        continue  # Skips to next step
    print(i)

Truthiness and Boolean Evaluation

if []:
    print("won't run")
if "0":
    print("runs")         # non-empty string is truthy

print(5 and 3)            # 3
print(0 or 9)             # 9

range() Semantics

range(5)          # 0..4
range(2, 7)       # 2..6
range(10, 0, -2)  # 10, 8, 6, 4, 2

# start + step*i must stay below stop (or above for negative step)

Loop else + State Tracing

for i in range(3):
    if i == 1:
        break
else:
    print("no break")   # skipped because break happened

for i in range(2):
    for j in range(2):
        break           # only inner loop breaks

Trace Example — write variable states per iteration to avoid mental skips:

i = 0
while i <= 3:
    i += 2
    print("*")   # prints two times

# trace:
# i=0 -> 2 (print)
# i=2 -> 4 (print)
# i=4 stops loop

Exam Strategy — Key Patterns

if/elif flow: only first true branch runs.
Loop tracing: exact effect of break, continue, and loop-else.
Range direction traps: wrong sign in step gives an empty range.
Placeholder statements: pass is valid syntax and often appears in unfinished branches.
Nested loops: break exits only the innermost active loop.
Loop-else rule: else runs only when loop finishes without break.
Boolean state tracing: short chains like x = x or y and y = x and y are frequent code-trace tasks.
Trace method: write variable states per iteration (i, counters, flags) to avoid mental skips.
Loop-else check: ask one question first: "Did any break execute?" If yes, skip loop-else.
Boolean check: evaluate left-to-right with short-circuit rules before simplifying expressions.
While-loop check: confirm that the update statement is still executed on every path (especially with continue).
Branch check: with if/elif/else, stop evaluating the chain after the first true condition.

📋 Cheatsheet: Control Flow Quick Reference

if/elif/else Syntax

if x > 10:
    print("Greater than 10")
elif x > 5:
    print("Greater than 5")
else:
    print("5 or less")

Comparison Operators

x == y      # Equality
x != y      # Inequality
x < y       # Less than
x > y       # Greater than
x <= y      # Less than or equal
x >= y      # Greater than or equal

Boolean Logic

x and y     # True if both are true
x or y      # True if at least one is true
not x       # Negation

# Chained comparison
1 < x < 10  # Equivalent to: (1 < x) and (x < 10)

for Loops

for i in range(5):
    print(i)  # 0, 1, 2, 3, 4

for i in range(2, 8):
    print(i)  # 2, 3, 4, 5, 6, 7

for i in range(0, 10, 2):
    print(i)  # 0, 2, 4, 6, 8 (step 2)

while Loops

count = 0
while count < 10:
    print(count)
    count += 1

# Without exit: infinite loop!
# while True:
#     print("Forever")

break and continue

for i in range(10):
    if i == 5:
        break       # Exits the loop
    if i % 2 == 0:
        continue    # Skips to next step
    print(i)

else with Loops

for i in range(5):
    if i == 3:
        break
else:
    print("Loop completed without break")

# The else is ONLY executed if the loop
# ends without break!

Truthy/Falsy in Conditions

Falsy: 0, 0.0, "", [], {}, (), None, False
Truthy: everything else (including "0" and [0])

Short-Circuit Cheats

False and expr   # expr not evaluated
True or expr     # expr not evaluated

5 and 8          # 8
0 and 8          # 0
5 or 8           # 5
0 or 8           # 8

⚠️ Common Mistakes in Control Flow

Mistake 1: Assignment instead of Comparison

Wrong:

if x = 5:  # SyntaxError!
    print("Yes")

Correct:

if x == 5:  # Comparison
    print("Yes")

Mistake 2: Missing Colon

Wrong:

if x > 5
    print("Greater")  # IndentationError!

Correct:

if x > 5:
    print("Greater")

Mistake 3: Misunderstanding range()

Problem: range(5) goes from 0 to 4, NOT to 5!

for i in range(5):
    print(i)  # Prints 0, 1, 2, 3, 4 (not 5!)

The upper bound is EXCLUSIVE!

Mistake 4: else in Loops Misunderstood

Problem: The else clause is only executed if the loop ends WITHOUT break!

for i in range(5):
    if i == 3:
        break
else:
    print("Done")  # This will NOT print!

The else is NOT executed because of break!

Mistake 5: Infinite Loops

Problem: A while loop without an exit condition runs forever!

x = 0
while x < 10:
    print(x)
    # Mistake: x never increases!
    # Loop runs forever...

Don't forget to update the loop variable!

Mistake 6: Expecting break to Exit All Loops

Trap: break exits only the innermost loop.

for i in range(2):
    for j in range(2):
        break
    print(i)  # still runs for each outer iteration

Mistake 7: Over-Comparing to True/False

Less Pythonic:

if flag == True:
    ...
if len(items) != 0:
    ...

Preferred:

if flag:
    ...
if items:
    ...

Mistake 8: Assuming continue Cancels Loop else

for i in range(3):
    if i == 1:
        continue
else:
    print("runs")  # runs, because no break

Mistake 9: Skipping Counter Updates in One Branch

Risk: accidental infinite loops.

i = 0
while i < 5:
    if i == 2:
        continue   # i never changes here
    i += 1

Fix:

i = 0
while i < 5:
    i += 1
    if i == 2:
        continue

🚨 Critical Exam Pitfalls

These are the most common traps that catch even experienced programmers in the PCEP exam. Master these to avoid losing easy points!

🚨 Loop else Clause Condition
❌ Wrong: else runs every time loop ends
✓ Correct: else runs only if NO break occurred
The else block in a loop only executes if the loop completes normally. If break is executed, the else is skipped.
🚨 range() Stop is Exclusive
❌ Wrong: range(5) = [0,1,2,3,4,5]
✓ Correct: range(5) = [0,1,2,3,4]
The stop value in range() is exclusive - it's not included. range(5) goes from 0 to 4, not 5.
🚨 Wrong Step Direction
❌ Wrong: range(5, 0, 1) returns [1,2,3,4]
✓ Correct: range(5, 0, 1) returns []
With a positive step, you can't go from 5 down to 0. Use a negative step: range(5, 0, -1) for [5,4,3,2,1].
🚨 break Only Exits Innermost Loop
❌ Wrong: break from nested loop exits all loops
✓ Correct: break only exits the innermost loop
A break statement only terminates the loop it's in. For nested loops, you need multiple breaks or a flag variable.
🚨 Chained Comparisons are Valid
❌ Wrong: 1 < x < 10 is not valid Python
✓ Correct: 1 < x < 10 is valid and equivalent to (1 < x) and (x < 10)
Python supports chained comparisons, which are both valid and more efficient than writing separate conditions.
🚨 Boolean Operators Return Values
❌ Wrong: 5 and 3 = True, 0 or 5 = True
✓ Correct: 5 and 3 = 3, 0 or 5 = 5
and and or return actual values, not booleans. and returns the first falsy value or the last, or returns the first truthy value or the last.
🚨 Assignment vs Comparison Operator
❌ Wrong: if x = 5: ...
✓ Correct: if x == 5: ... OR x = 5; if x: ...
Use == for comparison and = for assignment. In conditions, = is a syntax error in Python.
🚨 Loop Variable Persists After Loop
❌ Wrong: i is undefined after loop ends
✓ Correct: i remains accessible after loop with its final value
Loop variables don't have block scope in Python. The variable i continues to exist and holds the last value it had.
🚨 while True Without Break
❌ Wrong: while True continues until function ends
✓ Correct: while True creates infinite loop - must have break
A while True loop will run forever unless a break statement is executed. Always ensure an exit condition.
🚨 elif vs Multiple if Statements
❌ Wrong: All conditions are checked and all matching blocks run
✓ Correct: Only the first true condition executes with elif
With elif, only the first matching block executes. With multiple if statements, all conditions are checked and all matching blocks run.
🚨 all([]) and any([])
❌ Wrong: all([]) is False and any([]) is True
✓ Correct: all([]) is True and any([]) is False
Empty iterables are a common logic trap. Learn these two special cases for exam questions.
🚨 range() Is Not a List
❌ Wrong: type(range(5)) is list
✓ Correct: type(range(5)) is range
range() returns a range object (lazy sequence), not a concrete list.
🚨 Precedence in Mixed Boolean Conditions
❌ Wrong: and and or have equal precedence
✓ Correct: and binds tighter than or
Raw exam items repeatedly test this rule. Without parentheses, Python evaluates and before or.
🚨 continue Can Skip Critical Updates
❌ Wrong: continue never changes loop behavior significantly
✓ Correct: continue can skip updates and create infinite loops in while blocks
New imported items reinforce this: always check whether the counter update executes on every iteration path.
